When naming Craft Agent or orchestrator-owned sessions, use a short project/task-specific tag. `SVB` is only an example, not a fixed default. Orchestrator sessions use `[${tag}] Orchestrator (${project_name})`; spawned worker sessions use `${tag} ${title}`. Every non-orchestrator agent spawned by an orchestrator must have `subagent`, `project::<name>`, `status::in-progress`, `worktree::<name>` when applicable, and the primary role label matching its deliverable: `executor` for implementation, `auditor` for audit/review and plan audit, `designer` for product/UX/UI design, or `tester` for QA/testing/verification. Prefer exactly one primary role label. Load the matching canonical role skill: `craft-agent-executor`, `craft-agent-auditor`, `craft-agent-designer`, or `craft-agent-tester`; plan auditors load both `craft-agent-auditor` and `plan-auditor`. Spawn executor agents in Execute/allow-all by default when supported; audit/review agents may use safe/read-only modes. Spawned agents own their final state: preserve unrelated labels, replace `status::in-progress` with `status::done`, `status::blocked`, `status::error`, or `status::cancelled`, and set Craft session status to `needs-review` by default. Executor auto-close is allowed only when the prompt contains exactly `auto-close on success: true` and all verification passes; it never applies to audit/review agents or non-success outcomes. Orchestrators plan rather than implement, assess complexity 1–5, run the mandatory plan-audit gate, dispatch independent work in parallel, collect reports, and request result audits when confidence is below 95%. Complexity 1–2 requires one plan auditor; complexity 3 requires one auditor per round, complexity 4 two, complexity 5 three; complexity 3+ requires two rounds. Plan auditors do not implement. Treat OFFTOP/оффтоп/отп/ot/oft messages as ephemeral side checks. Peer orchestrators are not subagents. Preserve unrelated labels whenever updating `status::<name>`, `git::<status>`, or `worktree::<name>`.
