When naming Craft Agent or orchestrator-owned sessions, use a short project/task-specific tag in chat names. `SVB` is only an example tag derived from Servarium Backend, not a fixed default. Orchestrator sessions should be named `[${tag}] Orchestrator (${project_name})`. Spawned worker/audit sessions should be named `${tag} ${title}`. If the user specifies a tag, use it; otherwise infer a concise meaningful tag from the project/scope and keep it consistent across the orchestrator and workers. Spawn subagents in Execute/allow-all mode by default unless the user explicitly requests otherwise. Orchestrator agents should not implement directly: they plan, split work into independent tasks, dispatch agents, receive reports, and spawn audit/review agents when confidence in a worker result is below 95%. If a message starts with OFFTOP/оффтоп/отп/ot/oft, the orchestrator handles it directly as an ephemeral side check and must not add that information to durable plan/task/worker context. Orchestrators may spawn peer orchestrators, which are not subagents, when a new task arrives while current workers are still running and the new task would distract/conflict; peer orchestrator prompts must instruct use of a new worktree if the task may interfere with active work. If the new task is safe and non-conflicting, the current orchestrator should extend its own plan and dispatch agents as needed. For session labels, prefer valued labels `status::<name>`, `git::<status>`, and `worktree::<name>`; preserve unrelated labels when updating one label dimension.
